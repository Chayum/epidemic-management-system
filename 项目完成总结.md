# 项目完成总结

## 📋 项目概述

已成功完成 **面向防疫应急保障的微服务架构物资调度系统** 的全栈开发！

本项目采用 Spring Cloud Alibaba 微服务架构，实现了物资库存管理、物资申请审批、捐赠管理、疫情信息服务等功能，支持高并发场景下的物资调度需求。

```java

distributedLockUtil.executeWithLockWithRetry(lockKey, () -> {
    // 查询库存 -> 校验库存 -> 更新库存 -> 检查预警状态
```

#### 2. JWT + Redis双重验证
```java
// 1. JWT验证签名和过期时间
// 2. Redis检查黑名单
```

## 🚀 快速开始

### 环境要求

- JDK 17+
- MySQL 8.0+
- Redis 7.0+
- RabbitMQ 3.12+
- Nacos 2.3+
- Node.js 18+

### 后端启动

```bash
# 1. 初始化数据库
mysql -u root -p < backend/sql/epidemic_user.sql
mysql -u root -p < backend/sql/epidemic_material.sql
mysql -u root -p < backend/sql/epidemic_pandemic.sql

# 2. 启动Nacos（配置中心）
# 将 nacos/nacos 目录下的配置文件导入Nacos

# 3. 启动各微服务（按顺序）
# 网关服务
cd backend/epidemic-gateway && mvn spring-boot:run

# 用户服务
cd backend/epidemic-user-service && mvn spring-boot:run

# 物资服务
cd backend/epidemic-material-service && mvn spring-boot:run

# 疫情服务
cd backend/epidemic-pandemic-service && mvn spring-boot:run
```

### 前端启动

```bash
cd epidemic-material-management
npm install
npm run dev
```

### Docker部署

```bash
cd deploy
docker-compose up -d
```
