# 疫情防控物资调度管理系统 - 项目完成总结

## 一、项目概述

### 1.1 项目背景

本项目是一个基于 Spring Cloud 微服务架构的疫情防控物资调度管理系统，旨在解决疫情防控期间物资管理效率低、信息滞后、流程不规范等问题。

### 1.2 项目目标

- 实现物资全生命周期管理（捐赠→入库→申领→审批→发货→收货）
- 提供实时疫情信息发布与推送服务
- 支持多角色协同（管理员、申请方、捐赠方）
- 实现系统高可用、易部署

## 四、数据库设计

### 4.1 数据库分布

| 数据库 | 包含表 | 说明 |
|-------|-------|------|
| epidemic_user | sys_user, sys_operate_log | 用户与日志 |
| epidemic_material | material, stock_order, application, donation 等 | 物资相关 |
| epidemic_pandemic | pandemic_news, push_record, user_notification | 疫情信息 |

### 4.2 核心表结构

**物资表 (material)**
- 物资ID、名称、类型、规格、库存、预警阈值

**申请表 (application)**
- 申请单号、物资ID、数量、申请人、状态、审核信息

**捐赠表 (donation)**
- 捐赠单号、物资信息、捐赠方、审核状态

## 五、API 接口统计

### 5.1 接口概览

| 服务 | 接口数量 | 说明 |
|-----|---------|------|
| 用户服务 | 15+ | 登录注册、用户管理、日志查询 |
| 物资服务 | 30+ | 物资管理、库存操作、申请审批 |
| 疫情服务 | 10+ | 新闻管理、消息推送 |

### 5.2 主要接口

**用户服务**
- `POST /api/user/login` - 用户登录
- `GET /api/user/info` - 获取用户信息
- `GET /api/user/list` - 获取用户列表

**物资服务**
- `GET /api/material/list` - 物资列表
- `POST /api/application` - 提交申请
- `PUT /api/application/{id}/approve` - 审批申请
- `POST /api/donation` - 提交捐赠

**疫情服务**
- `GET /api/pandemic/news` - 新闻列表
- `POST /api/pandemic/push` - 发送推送

## 八、项目亮点

### 8.1 技术亮点

1. **微服务架构**：服务独立部署，便于扩展维护
2. **响应式前端**：Vue 3 组合式 API，代码更简洁
3. **Redis 深度应用**：
   - 分布式限流（全局/IP/用户三级限流）
   - 用户信息缓存（30分钟TTL）
   - 库存预警 Pub/Sub 发布订阅
   - API 接口缓存（仪表盘数据30秒TTL）
   - 分布式锁保障库存原子操作
4. **RabbitMQ 可靠消息**：
   - 消息重试机制（最多3次，间隔倍增）
   - 死信队列处理失败消息
   - 手动确认模式（ACK/NACK）
   - Redis 幂等性设计防止重复消费
5. **容器化部署**：Docker 一键部署，简化运维

### 8.2 业务亮点

1. **角色分离**：管理员、申请方、捐赠方三种角色，权限清晰
2. **流程完整**：覆盖物资捐赠到申领发放全流程
3. **状态追踪**：物流状态实时更新，全程可追溯
4. **数据可视化**：ECharts 图表展示统计数据
